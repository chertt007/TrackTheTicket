# System Flow (Source of Truth)

## Основной принцип

Каждая подписка проверяется по двум каналам:

- Channel A: Fast source (быстрый агрегатор/источник).
- Channel B: Direct airline (официальный сайт авиакомпании).

## Основной цикл

1. Пользователь создает подписку через Telegram.
2. Subscription Service сохраняет параметры мониторинга.
3. Оркестратор запускает проверки A и B.
4. Reconciliation сравнивает результаты.
5. Notification отправляет итог пользователю.

## Роль API

Все сервисные HTTP endpoint'ы реализуются на FastAPI.
Это единый стандарт для текущей и следующей итерации проекта.
